Extreme Programming (XP)

The Most Widely Used Agile Method

XP takes iterative development to the extreme. It is defined by a set of practices that, when used together, create a highly productive and flexible development environment.

Core Practices of XP

  • Incremental Planning: Requirements are recorded as 'user stories'. The customer prioritizes stories for each release.
  • Small Releases: The system is released frequently in small increments that add business value.
  • Simple Design: Just enough design is done to meet the current requirements‚Äîno more.
  • Test-first Development: Write tests before you write the code.
  • Refactoring: Continuously improve the code to keep it simple and maintainable.
  • Pair Programming: Developers work in pairs to write and review code in real-time.
  • Continuous Integration: Integrate new code into the system frequently, running all tests each time.
  • On-site Customer: A customer representative is available full-time to the team to answer questions and prioritize stories.
