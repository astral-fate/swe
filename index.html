Software Engineering Course Syllabus

This document outlines the core topics covered across the foundational chapters of Software Engineering, from initial concepts to agile development and requirements engineering.

Course Modules

Module Main Ideas & Topics Methodologies & Tools
Chapter 1: Introduction to Software Engineering
  • Generic vs. Customized software products.
  • The importance of professional software engineering.
  • Essential attributes of good software: Maintainability, Dependability, Efficiency, and Acceptability.
  • Fundamental activities: Specification, Development, Validation, and Evolution.
  • Diversity in software applications and key challenges like security and heterogeneity.
  • Professional and ethical responsibilities.
  • Web Services
  • Cloud Computing
Chapter 2: Software Processes
  • Abstract representations of software process models.
  • Plan-driven vs. Agile approaches.
  • The Waterfall Model: Sequential phases and limitations.
  • Incremental Development: Interleaved activities, benefits, and challenges.
  • Reuse-Oriented Engineering: Assembling systems from existing components.
  • Coping with changing requirements through prototyping and incremental delivery.
  • Waterfall Model
  • Spiral Model
  • Rational Unified Process (RUP)
  • UML
  • .NET / J2EE Frameworks
Chapter 3: Agile Software Development
  • Core values from the Agile Manifesto.
  • Principles: Customer involvement, incremental delivery, embracing change, and maintaining simplicity.
  • Extreme Programming (XP): An influential agile method.
  • Key XP practices: User stories, pair programming, test-first development, refactoring, and continuous integration.
  • Challenges in scaling agile methods for large, complex systems.
  • Extreme Programming (XP)
  • Scrum (mentioned as part of Agile Project Management)
  • Automated Test Frameworks (e.g., JUnit)
  • IDEs & GUI Toolsets
Chapter 4: Requirements Engineering
  • The process of establishing system services and constraints.
  • User vs. System requirements.
  • Functional Requirements: What the system should do.
  • Non-functional Requirements: Constraints on the system (e.g., performance, security).
  • The Software Requirements Document (SRD).
  • Core processes: Elicitation, Analysis, Validation, and Management.
  • Elicitation techniques like interviews, scenarios, and use cases.
  • UML Use Case Diagrams
  • UML Sequence Diagrams
  • Requirements Management Systems
  • Ethnography
